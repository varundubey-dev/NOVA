# NOVA Language Specification

### Version 0.3

NOVA v0.3 expands upon v0.2 by introducing:

* Arrays
* Typed Arrays
* Multi-Type Arrays
* Nested Arrays
* Array Access
* Array Mutation

---

# New Features

## Arrays

Arrays store ordered collections of values.

Arrays may contain:

* Primitive values
* Arrays

---

## Untyped Arrays

`[U]` represents an array capable of storing any valid NOVA value.

```text
items: [U] = [
    1,
    "hello",
    true,
    null
]
```

---

## Typed Arrays

Restrict elements to a single datatype.

```text
numbers: [N] = [
    1,
    2,
    3
]

names: [S] = [
    "A",
    "B",
    "C"
]
```

---

## Empty Arrays

```text
numbers: [N] = []

names: [S] = []

items: [U] = []
```

---

## Multi-Type Arrays

```text
mixed: [N,S] = [
    1,
    2,
    "hello"
]
```

Invalid:

```text
mixed: [N,S] = [
    true
]
```

Error:

```text
Invalid datatype inside array.
```

---

## Nested Arrays

```text
matrix: [[N]] = [
    [1,2,3],
    [4,5,6],
    [7,8,9]
]
```

---

## Array Access

Arrays use zero-based indexing.

```text
numbers[0]

numbers[1]
```

Nested access:

```text
matrix[0][1]
```

---

## Array Mutation

Mutable arrays may be modified.

```text
numbers: [N] = [
    1,
    2,
    3
]

numbers[0] = 10
```

Result:

```text
[10, 2, 3]
```

---

Datatype restrictions still apply.

```text
numbers[0] = "hello"
```

Error:

```text
Datatype mismatch.
```

---

## Immutable Arrays

Arrays declared as constants are immutable.

```text
numbers:: [N] = [
    1,
    2,
    3
]
```

Invalid:

```text
numbers[0] = 10
```

Error:

```text
Cannot modify immutable collection.
```
