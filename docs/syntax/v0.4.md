# NOVA Language Specification

### Version 0.4

NOVA v0.4 expands upon v0.3 by introducing:

* Map Schemas
* Map Instances
* Optional Properties
* Nested Schemas
* Schemas Containing Arrays
* Arrays Of Maps
* Property Access
* Property Mutation

---

# New Features

## Map Schemas

Map schemas define the structure and datatypes of map instances.

Schemas are declared using the `M` datatype.

```text
User: M = {
    name: S,
    age: N,
    isCoder: B
}
```

A schema does not store values.

It only defines structure.

---

## Schema Naming Rules

Schema names must begin with an uppercase letter.

Valid:

```text
User
Address
Product
EmployeeData
```

Invalid:

```text
user
address
product
employeeData
```

Rules:

* The first character must be an uppercase letter (`A-Z`)
* Remaining characters may contain letters (`a-z`, `A-Z`)
* Remaining characters may contain numbers (`0-9`)
* Remaining characters may contain underscores (`_`)
* Schema names may not start with a number
* Special characters are not allowed

---

## Map Instances

Variables may use schema names as datatypes.

```text
User: M = {
    name: S,
    age: N,
    isCoder: B
}

user: User = {
    name = "Varun",
    age = 19,
    isCoder = true
}
```

Every required property must be present.

Every value must match the schema datatype.

---

## Optional Properties

Optional properties use `?`.

```text
User: M = {
    name: S,
    age: N,
    nickname?: S
}
```

Valid:

```text
user: User = {
    name = "Varun",
    age = 19
}
```

Also valid:

```text
user: User = {
    name = "Varun",
    age = 19,
    nickname = "Nova"
}
```

---

## Nested Schemas

Schemas may contain other schemas.

```text
Address: M = {
    city: S,
    country: S
}

User: M = {
    name: S,
    address: Address
}
```

Usage:

```text
user: User = {
    name = "Varun",

    address = {
        city = "Mumbai",
        country = "India"
    }
}
```

---

## Schemas Containing Arrays

Schemas may contain typed arrays.

```text
User: M = {
    name: S,
    skills: [S]
}
```

Usage:

```text
user: User = {
    name = "Varun",

    skills = [
        "JavaScript",
        "TypeScript",
        "Python"
    ]
}
```

---

## Arrays Of Maps

Arrays may use schema types.

```text
User: M = {
    name: S,
    age: N
}

users: [User] = [
    {
        name = "Varun",
        age = 19
    },

    {
        name = "Nova",
        age = 21
    }
]
```

Every element must satisfy the `User` schema.

---

## Nested Arrays Of Maps

```text
User: M = {
    name: S,
    age: N
}

groups: [[User]] = [
    [
        {
            name = "Varun",
            age = 19
        }
    ],

    [
        {
            name = "Nova",
            age = 21
        }
    ]
]
```

Schema validation applies at every nesting level.

---

## Mixed Structures

Schemas may contain arrays, nested schemas, or both.

```text
Address: M = {
    city: S,
    country: S
}

User: M = {
    name: S,
    skills: [S],
    address: Address
}

users: [User] = [
    {
        name = "Varun",

        skills = [
            "JS",
            "TS"
        ],

        address = {
            city = "Mumbai",
            country = "India"
        }
    }
]
```

---

## Map Keys

Map property names follow the same naming rules as variables.

Valid:

```text
user_name
user1
first_name
```

Invalid:

```text
1user
user-name
user@name
```

Rules:

* Keys may contain letters (`a-z`, `A-Z`)
* Keys may contain numbers (`0-9`)
* Keys may contain underscores (`_`)
* Keys may not start with a number
* Special characters are not allowed

---

## Property Validation

All map instances are validated against their schema.

Missing required properties are not allowed.

Example:

```text
User: M = {
    name: S,
    age: N
}

user: User = {
    name = "Varun"
}
```

Error:

```text
Missing required property 'age'.
```

---

Unknown properties are not allowed.

Example:

```text
User: M = {
    name: S,
    age: N
}

user: User = {
    name = "Varun",
    age = 19,
    city = "Mumbai"
}
```

Error:

```text
Unknown property 'city'.
```

---

Datatype validation is enforced.

Example:

```text
User: M = {
    name: S,
    age: N
}

user: User = {
    name = "Varun",
    age = "19"
}
```

Error:

```text
Datatype mismatch.
```

---

## Property Access

Properties are accessed using dot notation.

```text
user.name

user.age
```

Nested access:

```text
user.address.city
```

Array access inside maps:

```text
user.skills[0]
```

Arrays of maps:

```text
users[0].name
```

Mixed access:

```text
users[0].address.city
```

---

## Property Mutation

Map properties may be modified if the map instance is mutable.

```text
user.name = "Nova"
```

Result:

```text
{
    name = "Nova",
    age = 19,
    isCoder = true
}
```

Datatype restrictions continue to apply.

```text
user.age = "twenty"
```

Error:

```text
Datatype mismatch.
```

Unknown properties may not be added.

```text
user.city = "Mumbai"
```

Error:

```text
Unknown property 'city'.
```

---

## Immutable Maps

Map instances declared as constants are immutable.

```text
user:: User = {
    name = "Varun",
    age = 19,
    isCoder = true
}
```

Invalid:

```text
user.name = "Nova"
```

Error:

```text
Cannot modify immutable collection.
```

Deep immutability applies to all nested arrays and maps contained within a constant collection.

---

## Generic Maps

Generic maps are not supported.

Invalid:

```text
user: M
```

Invalid:

```text
users: [M]
```

Map instances must use a declared schema type.

Valid:

```text
user: User

users: [User]
```
